# 許願池後端串接說明（資料用 Google Sheet）

許願池的「資料」可以放在 Google 試算表（Sheet），「圖片」則不適合直接存在 Sheet 裡，需要另外存成**圖片網址**再寫進 Sheet。以下說明怎麼串接。

---

## 一、Google Sheet 欄位建議（顧客輸入的欄位集中寫入同一張 Sheet）

在試算表**第一列**設為標題，建議欄位如下。**顧客在許願池表單輸入的所有欄位，後端都應寫入同一張 Sheet 的同一列**，對照如下：

| Sheet 欄位 | 對應表單欄位 | 說明 | 範例 |
|------------|--------------|------|------|
| id | （後端自動） | 許願編號（唯一） | 1 |
| title | 想要什麼商品（必填） | 商品名稱／描述 | 吉伊卡哇鯛魚燒 |
| note | 規格／備註 | 尺寸、數量、顏色等 | 尺寸 M |
| category | 商品類別 | 下拉選單 | 角色周邊 |
| link | 商品連結 | 網址 | https://... |
| region | 地區／店家 | 文字 | 東京、大阪 |
| status | （後端預設） | 狀態 | 許願中 |
| image1 | 圖片網址 1（後端上傳後寫入或手動補） | 主圖 | https://... |
| image2 | 圖片網址 2 | （可留空） | |
| image3 | 圖片網址 3 | （可留空） | |
| supportCount | （後端預設） | 集氣人數 | 0 |
| createdAt | （後端自動） | 建立時間 | 2026-03-01 12:00 |

- **顧客填的欄位**：title、note、category、link、region 全部由後端收到後寫入 Sheet 對應欄位。
- 前端會用 **image1** 當卡片主圖；若沒有 image1，會顯示 emoji。
- **圖片欄位一律存「網址」**；若後端有接收上傳圖，再寫入 image1～image3。

### Sheet 欄位範例（直接複製到試算表第一列）

**第 1 列（標題列）**，從 A1 開始橫向貼上：

| A | B | C | D | E | F | G | H | I | J | K | L |
|---|----|----|----|-----|--------|--------|--------|--------|--------|--------------|----------|
| id | title | note | category | link | region | status | image1 | image2 | image3 | supportCount | createdAt |

**純文字版（可整行複製到試算表）：**

```
id	title	note	category	link	region	status	image1	image2	image3	supportCount	createdAt
```

**第 2 列起為資料範例（一筆）：**

| A | B | C | D | E | F | G | H | I | J | K | L |
|---|----|----|----|-----|--------|--------|--------|--------|--------|--------------|----------|
| 1 | 吉伊卡哇鯛魚燒 | 尺寸 M | 角色周邊 | https://example.com/item | 東京 | 許願中 | https://drive.google.com/uc?export=view&id=xxx | （留空） | （留空） | 15 | 2026-03-13 14:00 |

- **id**：數字，每筆唯一（後端 doPost 可依列號自動給）。
- **title**～**region**：對應顧客表單欄位。
- **status**：預設「許願中」；可手動改為「已達標」「已下架」等。
- **image1**～**image3**：圖片網址，留空則卡片顯示 emoji。
- **supportCount**：集氣人數，新許願可寫 0。
- **createdAt**：建立時間（例：2026-03-13 14:00）。

---

## 二、顧客貼圖片那邊，只能自動寫入 Sheet 嗎？

**不是只能自動寫入 Sheet**，而且 Sheet 本身**不能存圖片檔案**，只能存「圖片網址」。所以有幾種做法：

| 做法 | 說明 |
|------|------|
| **A. 自動寫入「網址」到 Sheet** | 顧客在表單上傳圖片 → 你的**後端**接收檔案 → 後端把圖存到 Google Drive（或圖床）→ 取得圖片網址 → 後端把該**網址**寫入 Sheet 的 image1 / image2 / image3。這樣就是「顧客貼圖 → 自動在 Sheet 出現對應的圖片網址」。 |
| **B. 不自動存圖，改請顧客貼「圖片連結」** | 表單不加「上傳檔案」，改成一個欄位「圖片連結（選填）」；顧客自己把圖傳到 Imgur、Google Drive 等取得連結後貼上。後端只把這串**網址**寫入 Sheet，沒有「自動存圖」的步驟。 |
| **C. 手動補網址** | 顧客送出表單時只送文字（不送圖），你之後在 Sheet 裡手動把圖片網址貼進 image1 等欄位。 |

所以：**若希望「顧客一貼圖就自動在 Sheet 裡有那張圖的網址」**，就必須有後端（例如 Google Apps Script + Drive）負責：接收上傳 → 存圖 → 取得 URL → 寫入 Sheet。  
若不想做後端上傳，可以改用「請顧客貼圖片連結」或你手動補網址。

---

## 三、圖片要放哪裡？怎麼取得網址？

Sheet 只能存文字與數字，**不能直接存圖片檔案**，所以要先把圖片上傳到某個會「給網址」的地方，再把網址貼進 Sheet。

### 做法 A：Google 雲端硬碟（推薦）

1. 開一個資料夾專門放許願池圖片（例如「許願池圖片」）。
2. 上傳圖片後，對該檔案按**右鍵 → 取得連結**。
3. 將連結權限設為「**知道連結的任何人**」可檢視。
4. 連結會像：`https://drive.google.com/file/d/檔案ID/view?usp=sharing`  
   要給網頁當 `<img src="">` 使用，請改成：  
   `https://drive.google.com/uc?export=view&id=檔案ID`  
   （只保留中間的「檔案ID」替換進去即可。）

把這個「可直接顯示的圖片網址」貼進 Sheet 的 **image1**（或 image2、image3）。

### 做法 B：後端 API 代為上傳

- 使用者在前端選圖片後，由**你的後端**接收檔案、上傳到 Google Drive（或 Imgur 等圖床），取得圖片網址。
- 後端再把該網址寫入 Sheet 對應欄位。
- 這樣「上傳」與「寫入 Sheet」都在後端完成，你只要在 Sheet 裡維護「網址」即可。

## 四、用 Google Apps Script 把 Sheet 當後端

不用自己架伺服器，用 **Google Apps Script（GAS）** 綁在試算表上，就能：

- **讀取**：把 Sheet 資料轉成 JSON，給網頁用（許願列表）。
- **寫入**：接收表單送出（含圖片網址或上傳後的網址），寫回 Sheet。

### 4.1 建立試算表與 GAS

1. 新增一個 Google 試算表，第一列照上面欄位打標題。
2. 選單 **擴充功能 → Apps Script**，開新專案。
3. 在專案裡新增一個腳本（例如 `WishPoolAPI`），貼上下面的程式碼。

### 4.2 讀取許願列表（GET → 回傳 JSON）

```javascript
function doGet(e) {
  var sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  var data = sheet.getDataRange().getValues();
  var headers = data[0];
  var rows = data.slice(1);
  var list = [];
  for (var i = 0; i < rows.length; i++) {
    var row = rows[i];
    var obj = {};
    for (var j = 0; j < headers.length; j++) {
      obj[headers[j]] = row[j] != null ? row[j] : "";
    }
    list.push(obj);
  }
  return ContentService
    .createTextOutput(JSON.stringify({ wishes: list }))
    .setMimeType(ContentService.MimeType.JSON);
}
```

- 部署為 **網路應用程式**：執行身分「我」、存取權「所有使用者」（或「任何人」），取得**部署網址**。
- 前端用這個網址當 **WISH_LIST_URL**，用 GET 去要許願列表，回傳格式為：`{ "wishes": [ { "id": "1", "title": "...", "image1": "https://...", ... } ] }`。

### 4.3 接收許願表單（POST）＋**所有欄位寫入 Sheet**

前端會以 **JSON** 送出顧客填寫的欄位：`title`、`note`、`category`、`link`、`region`。後端收到後，**一次寫入 Sheet 一筆新列**，欄位對照見上一節。

**doPost 範例：接收 JSON，把顧客輸入的所有欄位集中寫入 Sheet**

```javascript
function doPost(e) {
  try {
    var json = JSON.parse(e.postData.contents);
    var sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
    var lastRow = sheet.getLastRow();
    var newId = lastRow; // 或改用 getNextId()
    var now = Utilities.formatDate(new Date(), "Asia/Taipei", "yyyy-MM-dd HH:mm");

    var row = [
      newId,
      json.title || "",
      json.note || "",
      json.category || "其他",
      json.link || "",
      json.region || "",
      "許願中",
      "",  // image1
      "",  // image2
      "",  // image3
      0,   // supportCount
      now  // createdAt
    ];
    sheet.appendRow(row);

    return ContentService.createTextOutput(JSON.stringify({ ok: true, id: newId }))
      .setMimeType(ContentService.MimeType.JSON);
  } catch (err) {
    return ContentService.createTextOutput(JSON.stringify({ ok: false, error: err.toString() }))
      .setMimeType(ContentService.MimeType.JSON);
  }
}
```

**重要**：Sheet 的**欄位順序**要與上面 `row` 陣列一致（第 1 欄 id、第 2 欄 title、第 3 欄 note…），否則請依你的標題列調整 `row` 的順序。

**圖片怎麼處理**

- **方式一**：先不送圖，顧客只送文字；image1～image3 留空，之後你在 Sheet 手動貼圖片網址。
- **方式二**：若後端有「上傳圖片 → 存 Drive → 回傳 URL」的 API，可再讓前端先上傳圖、取得 URL 後放進 JSON 的 image1 / image2 / image3，再由 doPost 一併寫入 Sheet。

## 五、前端要設定的兩個網址

在 `wish-pool.html` 的 script 裡有兩個變數：

| 變數 | 用途 | 範例 |
|------|------|------|
| **WISH_LIST_URL** | 讀取許願列表（GET），回傳 JSON | 你的 GAS 部署網址（GET） |
| **WISH_SUBMIT_URL** | 送出許願表單（POST） | 同一支 GAS 的網址（POST 時用 doPost） |

- **資料用 Sheet**：把 GAS 的「讀取」網址填進 `WISH_LIST_URL`，頁面載入時就會用 API 取得許願列表並渲染卡片。
- **圖片**：在 Sheet 的 image1 / image2 / image3 存**圖片網址**（例如 Google Drive 的 `https://drive.google.com/uc?export=view&id=xxx`），前端會用 image1 當卡片主圖。

## 六、API 回傳格式（給前端用）

**GET 許願列表** 建議回傳：

```json
{
  "wishes": [
    {
      "id": "1",
      "title": "吉伊卡哇鯛魚燒",
      "note": "",
      "category": "角色周邊",
      "status": "許願中",
      "image1": "https://drive.google.com/uc?export=view&id=你的檔案ID",
      "image2": "",
      "image3": "",
      "link": "",
      "region": "",
      "supportCount": "15"
    }
  ]
}
```

- 有 **image1** 時，卡片會顯示該圖片；沒有則顯示 emoji 或預設樣式。
- **supportCount** 會用來顯示集氣進度（例如 15/50）。

**POST 集氣 (addSupport)**：顧客按「我也想要」時，前端會對同一個 `WISH_SUBMIT_URL` 送 POST，body 為 `{ "action": "addSupport", "wishId": "1" }`。後端需依 wishId 找到該列，將試算表該列的 **supportCount 欄位 +1** 並寫回，回傳 `{ "ok": true, "supportCount": 16 }`。這樣集氣數字會寫入試算表，所有人看到的數字一致。

## 七、總結

| 項目 | 做法 |
|------|------|
| **顧客輸入的欄位** | 表單欄位 title、note、category、link、region 會以 **JSON** 送出，後端 **doPost 一次寫入 Sheet 一筆新列**，集中存在同一張試算表。 |
| **資料** | 用 Google Sheet 存，透過 Apps Script 提供 GET（讀列表）與 POST（寫入新許願，含上述所有欄位）。 |
| **圖片** | 不存進 Sheet，只存「圖片網址」。image1～image3 可先留空，之後手動貼網址，或由後端上傳圖後寫入。 |
| **前端** | 設定 `WISH_LIST_URL` 與 `WISH_SUBMIT_URL`，列表從 API 載入；送出許願時會送出所有欄位的 JSON；顧客按「集氣」時會呼叫 addSupport API，將 supportCount 寫回試算表。 |

## 八、測試時沒寫入 Sheet／許願池沒畫面？

依序檢查：

1. **試算表第一列必須是標題**  
   從 A1 開始要有：`id`、`title`、`note`、`category`、`link`、`region`、`status`、`image1`、`image2`、`image3`、`supportCount`、`createdAt`（可複製前面「Sheet 欄位範例」）。

2. **GAS 部署設定**  
   - 擴充功能 → Apps Script → 貼上 `許願池API.gs` 後**儲存**。  
   - **部署 → 新增部署** → 類型選「網路應用程式」→ **執行身分：我**、**存取權：任何人**（或「知道連結的任何人」）。  
   - 複製產生的**網址**，貼到 `wish-pool.html` 的 **WISH_LIST_URL** 與 **WISH_SUBMIT_URL**（在檔案開頭 `<script>` 內、註解「【必填】API 網址」下方，兩行都要貼同一個網址）。

3. **第一次要先授權**  
   在瀏覽器**新分頁**直接開啟該 GAS 網址，登入 Google 並按「允許」，完成一次授權後，前端再呼叫才會正常。

4. **CORS（跨網域）**  
   若許願池網頁和 GAS 不同網域（例如用 `file://` 開本機檔、或放在 GitHub Pages），瀏覽器可能擋請求。  
   - 前端已支援 **JSONP**：GET 失敗時會自動改用 `?callback=...` 載入列表。  
   - POST 送出許願若被擋，可暫時用**同網域**的網頁測試（例如把許願池放進同一個可對外開的站），或日後用後端 proxy 轉發。

5. **沒畫面**  
   - 若沒填 `WISH_LIST_URL`，會顯示**預設三張範例卡片**。  
   - 若有填但載入失敗，會保留預設卡片，並顯示黃色提示「無法從伺服器載入許願列表…」。  
   - 若整頁空白，請按 F12 開開發者工具看 **Console** 是否有紅色錯誤。

6. **沒寫入 Sheet**  
   - 確認 `WISH_SUBMIT_URL` 和 GAS 網址一致、且已部署為「任何人」存取。  
   - 送出許願後到試算表**重新整理**，新列會出現在最下面。  
   - 若仍沒有，到 GAS 編輯器左側**執行作業**查看是否有錯誤。

7. **出現「送出失敗，請稍後再試或聯絡闆闆」**  
   - **先完成授權**：用瀏覽器**新分頁**開啟你的 GAS 部署網址（和 WISH_SUBMIT_URL 同一個），登入 Google 後按「允許」或「進階 → 前往…」完成授權。完成後再回許願池頁面重試送出。  
   - **確認部署**：Apps Script → 部署 → 確認類型為「網路應用程式」、存取權為「**任何人**」（不是「僅限我自己」）。  
   - **跨網域（CORS）**：若許願池用 `file://` 或放在其他網域，瀏覽器可能阻擋 POST。前端已改為以 `text/plain` 送出以減少被擋機率；若仍失敗，可改將許願池頁面放到與 GAS 同網域，或使用可轉發請求的伺服器。
